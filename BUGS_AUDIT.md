### [security] WebSocket JWT ticket transmitted in URL query string — logged by proxies and servers

- **File:** `backend/api/src/domains/ws.rs:19-23`  ·  confidence 0.95

- **What:** The WebSocket upgrade is authenticated via a `token=<jwt>` query parameter (`GET /api/v1/ws?token=...`). Query strings appear in: HTTP server access logs, reverse-proxy (nginx/Cloudflare) logs, CDN logs, browser history, and the `Referer` header of any subsequent request. Although the ticket is short-lived (120 s), this TTL window is sufficient for a log-scraping attacker to replay it against any REST endpoint (compounded by finding #2 — aud is not enforced on REST routes).
